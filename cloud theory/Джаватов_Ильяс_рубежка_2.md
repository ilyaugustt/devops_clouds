# Вариант 1 Апрель 2020 года, ваша команда выиграла дорогостоящий тендер на разработку системы пропусков во время карантина. Предложите сервисы в облаке, которые могло бы использовать такое приложение. Можно на конкретных примерах провайдеров (AWS, Azure, Google, Yandex,Mail.RU).

Для начала - это вычислительные ресурсы. Нельзя заранее предсказать, сколько заявок прилетит в первые дни. Их может быть безгранично много. Поэтому нужны сервисы с возможностью автоматического масштабирования. Например, в AWS есть EC2 с авто-масштабированием, в Azure — Virtual Machine Scale Sets, в Яндекс.Облаке — Compute Cloud с управлением группами ВМ.

Хранение данных. Тут можно использовать реляционную базу данных как сервис: в AWS — RDS, в Azure — SQL Database, в Google Cloud — Cloud SQL, а в Яндекс.Облаке — Managed PostgreSQL.

API. Здесь пригодятся сервисы по управлению API-шлюзами: API Gateway в AWS или Яндекс, Azure API Management. Это позволит не городить всё вручную и сразу получить готовый механизм для приёма запросов, контроля доступа и логирования.

Безопасность. Мы ведь работаем с персональными данными, а это всегда риск. Тут нужны сервисы для шифрования и управления ключами: например, AWS KMS, Azure Key Vault, Yandex Lockbox. Также можно подключить двухфакторную аутентификацию через Cognito (AWS).

Аналитика. Точно не помешало бы в реальном времени отслеживать, сколько заявок подано, где возникают ошибки. Здесь пригодятся сервисы вроде AWS CloudWatch, Azure Monitor или Яндекс Monitoring.

В итоге получается, что облако закрывает все основные потребности: масштабирование, хранение, безопасность, API и мониторинг. Облако дало бы возможность развернуть систему буквально за несколько дней и сразу под огромную нагрузку.
